# MathReels - proof of concept

A "math reels" web app for "myself": a TikTok-style vertical feed of NSW ATAR
exam questions where students don't just pick an answer - they **walk through the
solution** one step at a time. Think *ChessReels, but for HSC maths*.

Built as a real, runnable stack you can open in VSCode:

- **Frontend** - React + Vite, full-screen swipeable reels, math rendered with KaTeX
- **Backend** - Node + Express + SQLite (answers checked server-side, attempts logged)

---

## What's in the demo

**Courses (filterable):** Standard · Advanced · Extension 1 · Extension 2 · Years 11 & 12
**15 seed questions** covering all four *interaction mechanics* you asked to see:

| Mechanic | Where to find it |
|---|---|
| **MCQ "what's the next step?"** | Logarithms (Advanced), Domain (Advanced), Arrangements (Ext 1) |
| **Type the first step** (free text, forgiving matching) | Gradient (Standard), Factorise (Advanced), Magnitude (Ext 1) |
| **Walk-through solution** (MCQ → text, multi-step) | Differentiation (Advanced), Remainder theorem (Ext 1), Induction (Ext 1), Complex numbers (Ext 2) |
| **Scratchpad** (rough-work canvas, works with finger/stylus) | Toggle on any question flagged with rough work |

Answer matching is forgiving: `√2`, `sqrt(2)`, `rt2`, `1.41` all pass; `(x-2)(x-3)`
and `(x-3)(x-2)` both pass. All checking happens on the **server** - the browser never
receives the answer key.

### Every reel names one key concept

Each question is tagged with the single named idea it drills (e.g. "The power rule",
"The sine rule", "Factorials: n!"). It shows as a chip on the idea screen and again
under the question, so the student always knows what one thing this reel is about.

### Teach the idea, then test it (ChessReels-style)

Some reels open with a **concept screen** before the question - the student sees the
*idea* first, then applies it. A concept screen can include any of:

- An **animated canvas** for ideas that are best seen in motion. Two ship so far:
  - **Tangent / derivative** (on the *Gradient at a point* reel): a secant line slides
    until two points merge and it becomes the tangent, the gradient at that point.
  - **Area / integral** (on the *Definite integral* reel): rectangles get thinner and
    more numerous until they fill the area exactly, the definite integral.
- A **quick worked example** of a *similar* question, revealed one line at a time, so
  the student follows the reasoning before trying the real one themselves.

For some topics, showing the method **applied** beats showing intuition. The **product
rule** reel (*Differentiate a product*) does exactly this: no animation, just a worked
`y = x^2 e^x` solved step by step, then the student applies the same pattern to
`y = x^3 \sin x`.

Captions narrate each animation stage, and there's play/pause + replay (captions scale
to however many you write). Add a new animation by writing a `draw` function in
`frontend/src/components/ConceptAnimation.jsx`, registering it under a `kind`, and adding
a `concept` object to a question in `seed.js`. A concept object looks like:

```js
concept: {
  kind: "tangent",            // omit for no animation (worked-example-only screens)
  title: "The idea: ...",
  captions: ["...", "..."],   // any length; omit/empty for no animation
  takeaway: "One-line summary, supports $LaTeX$.",
  worked: {                   // optional quick example of a similar question
    title: "Quick example first",
    problem: "Differentiate $y = x^2 e^x$.",
    steps: ["line 1", "line 2", "..."]
  }
}
```

## Run it locally (two terminals)

You'll need **Node 18+** installed. In VSCode, open this folder and use a split terminal.

### 1 - Backend (port 4000)
```bash
cd backend
npm install
npm run seed     # creates mathreels.db and loads the questions
npm start        # → http://localhost:4000
```

### 2 - Frontend (port 5173)
```bash
cd frontend
npm install
npm run dev      # → http://localhost:5173
```

Open **http://localhost:5173**. The Vite dev server proxies `/api` to the backend,
so there's nothing else to configure. For the best feel, open your browser dev tools
and toggle the mobile/device view - it's designed phone-first.

> Tip: `npm run dev` in the backend folder restarts the API on file changes.

## How it's structured

```
math-reels/
├── backend/
│   ├── server.js   # Express API + server-side answer checking
│   ├── db.js       # SQLite schema (questions, attempts)
│   ├── seed.js     # the NSW ATAR question bank  ← edit me to add questions
│   └── mathreels.db (generated by `npm run seed`)
└── frontend/
    └── src/
        ├── App.jsx              # feed + course/year filters + stats
        ├── components/
        │   ├── Reel.jsx         # one question, step-by-step state machine
        │   ├── ConceptAnimation.jsx # canvas animations (tangent, area)
        │   ├── WorkedExample.jsx    # "watch me solve a similar one", line by line
        │   ├── MathText.jsx     # renders text with inline $LaTeX$
        │   └── Scratchpad.jsx   # rough-work canvas
        └── styles/app.css
```

### API
| Method | Route | Purpose |
|---|---|---|
| GET | `/api/questions?course=&year=` | the reel feed (answer keys stripped) |
| POST | `/api/check` | check one step's answer + log the attempt |
| GET | `/api/stats` | solved count / accuracy for the header |
| GET | `/api/courses` | distinct course list |

## Adding your resource bank

Each question lives in `backend/seed.js` as an object with an ordered `steps` array.
A step is either `kind: "mcq"` (with `options` + `correctIndex`) or `kind: "text"`
(with an `accepted` list). Mixing them gives the "first step MCQ, then walk me through
it" flow. Add objects, run `npm run seed` again, refresh - that's the whole loop.

When the proof of concept gets the green light, the same schema scales straight to your
large resource bank (swap SQLite for Postgres, add topics/sub-strands, tagging by
upcoming exam, spaced-repetition ordering, etc.).
